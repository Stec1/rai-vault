# RAi Vault

RAi Vault is the project's **structured operating memory**: a markdown-first knowledge system for decisions, execution context, prompts, audits, and handoffs.

## What this vault is
- Founder operating system for planning and continuity.
- Long-term memory layer for AI-assisted execution.
- Decision, audit, and issue execution archive.

## What this vault is not
- Not the RAi product/application codebase.
- Not a deployment target.
- Not a substitute for implementation truth.

## Vault vs Product Repo
- **Product repo**: source of truth for implementation, APIs, infra code, migrations, and runtime behavior.
- **Vault repo**: source of truth for operating context, plans, prompts, decisions, and execution memory.

## How to use this vault
1. Start in [[00_MOC_RAi]] for navigation.
2. Check [[00_Current_State]] and [[01_Next_Action]] before any execution.
3. For active work, open the relevant `40_Execution/ISSUE-XX` folder.
4. Log major choices in [[Decision_Index]].
5. Store reusable prompts in `50_Prompts`.

## Where to find key information
- Current state: [[00_Current_State]]
- Next action: [[01_Next_Action]]
- Prompts: [[04_MOC_Prompts]] and `50_Prompts/`
- Decisions: [[Decision_Index]] and `70_Decisions/`
- Issue execution memory: `40_Execution/ISSUE-XX/`
