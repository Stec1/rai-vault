# DL-27 — PNG Domain Objects Removed from MVP Visual System

## Status
Accepted

## Decision
PNG domain objects are removed from the MVP visual system as canonical domain representation. Domain visualization is centered on `/explore` SVG topology; PNG-based domain showcases are not part of active MVP UI direction.

## Why
- PNG object direction introduced low-fidelity visual perception relative to target product quality.
- Keeping one canonical domain visual language reduces UI fragmentation.

## Trade-off
- Simplifies and unifies MVP visual direction.
- Defers alternative domain visual treatments outside `/explore`.

## Revisit
Yes. Revisit if future design system work introduces a better multi-surface domain representation strategy.
