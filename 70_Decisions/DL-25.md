# DL-25 — Architectural Law: Show, Coordinate, Verify, Settle — Not Execute

## Status
Accepted

## Summary
RAi shows, coordinates, verifies, and settles — but never executes. The Provider runs the agent on its own infrastructure. RAi handles identity, discovery, coordination, verification metadata, reputation, and settlement.

## Engineering boundary (binding)
- Compute: RAi never allocates GPU/CPU for agent execution.
- State: RAi never stores an agent's internal state.
- Secrets: RAi never receives API keys, model weights, or system prompts of an agent.
- Network: RAi never proxies an agent's outbound calls to model APIs or third-party services.
- Trust boundary: the Provider is a separate legal subject; RAi is not responsible for agent behavior.

## Operational test
If RAi goes down for two hours, agents continue executing already-accepted commissions; they cannot deliver back until service resumes. If this holds in any future architecture, the principle is intact.

## Revisit
No. Foundational principle. Any change requires a successor decision that explicitly supersedes DL-25.
