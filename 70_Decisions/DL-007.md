# DL-007: Lazy Mirror Policy for Repo Decisions in Vault

## Status
Accepted

## Context
The RAi product repo maintains its own decision log at `docs/decision-log.md` (DL-01 through DL-23). The vault maintains its own operating-level decision index at `70_Decisions/Decision_Index.md` (DL-001, DL-002, ...). Product and architecture notes in the vault occasionally need to reference repo decisions (for example, `28_Reputation_Model` references DL-22). Without a mirror policy, such references break the vault link graph or drift over time.

## Decision
Adopt a **lazy mirror policy**:
- Repo decisions are NOT mirrored into the vault by default.
- A repo decision is mirrored into the vault ONLY when at least one vault note needs to link to it.
- Mirrored vault notes use the same ID as in repo (e.g. DL-22) and live in `70_Decisions/DL-22.md`.
- Vault-side mirror is a summary, not a full copy. It links back to `docs/decision-log.md` for the authoritative version.
- If the repo decision changes, the vault mirror is updated in the same session the change is observed.

## Consequences
- Identifier convention: vault-native decisions use zero-padded 3-digit IDs (DL-001, DL-007), while repo-mirrored decisions retain repo-style non-padded IDs (DL-22); therefore DL-007 and DL-07 are intentionally different identifiers.
- Broken wiki-links to repo decisions are avoided.
- Vault does not get bloated with 20+ mirror files when only a handful are referenced.
- `Decision_Index.md` may contain both vault-native IDs (DL-001...) and mirrored repo IDs (DL-22, etc.) — this is intentional.
- A vault-side mirror note must always indicate its repo origin in the first line.
