# DL-008: Approve SI as Target Architecture (Post-MVP)

## Status
Accepted

## Context
The RAi product reached a point where a long-term post-MVP destination needed to be formalized to align documentation, decisions, and operating memory. Founder review concluded that the System Intelligence (SI) model — as captured in `docs/si-target.md` in the RAi product repo — is the chosen north-star architecture for Phase 3+.

## Decision
Accept the SI target architecture defined in `docs/si-target.md` as the canonical post-MVP destination of the RAi platform.

Active MVP execution remains governed by `docs/mvp-contract.md`, `ROADMAP.md`, and the current issue sequence. This acceptance does not modify MVP scope, active implementation work, or completed issue memory.

## Consequences
- The vault now anchors a target-state reference: `30_Architecture/38_SI_Target_Bridge.md`.
- The architectural law from repo DL-25 is mirrored as `70_Decisions/DL-25.md` under the lazy-mirror policy.
- No completed or active ISSUE folder is modified.
- No `40_Execution/01_Next_Action.md` change.
- MVP scope is not expanded.
- Bridge mapping in `docs/mvp-to-si-bridge.md` is descriptive only.
- New vault notes referencing target architecture must link to `docs/si-target.md` in the repo and to [[38_SI_Target_Bridge]] in the vault, not invent parallel definitions.

## Current Execution State
- ISSUE-00 through ISSUE-06 are completed in the product repo.
- ISSUE-06 vault closure remains a separate future pass.
- Start Page refinement and domain visual asset work remain separate from this SI memory anchoring pass.
