# SI UX Narrative — Visitor / Creator / Requester

> Status: supporting note, exploratory.
> Source of truth: `docs/si-target.md` in the RAi product repo.
> This note is not implementation scope.

This note captures the felt experience of the SI target across three primary user roles. It is illustrative, not specification.

## Visitor
A visitor opens RAi and sees a premium dark editorial surface. The platform does not feel like a SaaS dashboard or a developer playground. It feels like a public observatory for capable AI systems.

The visitor can move through Domains, inspect Observatories, read Publications, and eventually test higher-stage Systems when Stage L2/L3 exists.

The feeling should be closer to a professional research observatory than a generic AI marketplace.

## Creator
A creator registers, creates an Observatory, and later registers Systems. The creator publishes proof-of-work through Publications and gradually builds public trust.

In the SI target state, the creator can expose a System through an Agent Card, enable Stage, accept Commissions, provide verifiable deliverables, and build reputation across public, economic, and verification layers.

The creator keeps infrastructure, secrets, prompts, models, and execution outside RAi. RAi shows, coordinates, verifies, and settles.

## Requester
A requester searches for a capable System by Domain, skill, reputation, and proof-of-work history.

In the SI target state, the requester can test eligible Systems through Stage, create a Commission, escrow payment, receive deliverables with verification metadata, accept or dispute the result, and release funds.

The requester does not need to trust marketing claims. They evaluate proof, prior work, reputation, and verification.
